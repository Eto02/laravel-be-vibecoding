Perform a comprehensive API standards audit on this project.

## Step 1: Discover All Routes

Run:
```bash
php artisan route:list --path=api --json
```

Parse the JSON output to get all routes with their method, URI, controller, and action.

## Step 2: Check for ApiResponse Blocker

First, verify whether `app/Http/Responses/ApiResponse.php` exists. If it does NOT exist:

```
BLOCKER: app/Http/Responses/ApiResponse.php does not exist.
All controllers that reference ApiResponse will fail at runtime.
This must be created before any other work can proceed.
```

Show this implementation to create it:

```php
<?php

namespace App\Http\Responses;

use Illuminate\Http\JsonResponse;

class ApiResponse
{
    public static function success(
        string $message,
        mixed $data = null,
        int $status = 200,
        array $extraMeta = [],
    ): JsonResponse {
        return response()->json([
            'success' => true,
            'message' => $message,
            'data'    => $data,
            'meta'    => array_merge(['timestamp' => now()->toISOString()], $extraMeta),
        ], $status);
    }

    public static function error(
        string $message,
        int $status = 400,
        array $errors = [],
    ): JsonResponse {
        $body = [
            'success' => false,
            'message' => $message,
            'meta'    => ['timestamp' => now()->toISOString()],
        ];
        if (!empty($errors)) {
            $body['errors'] = $errors;
        }
        return response()->json($body, $status);
    }

    public static function validationError(string $message, array $errors): JsonResponse
    {
        return self::error($message, 422, $errors);
    }
}
```

## Step 3: Inspect Each Controller

For each controller file under `app/Http/Controllers/Api/`, read the file and evaluate 5 criteria:

**Check A — Response Envelope**
- PASS: Uses `ApiResponse::success()` or `ApiResponse::error()`
- WARN: Uses `response()->json([...])` manually (legacy, needs migration)
- FAIL: Returns raw array, model, or Collection without any wrapping

**Check B — FormRequest Usage**
- PASS: Method signature contains a class that extends `FormRequest`
- WARN: Uses `$request->validate([...])` inline
- FAIL: Uses `Validator::make(...)` inside the method body

**Check C — Resource Wrapping**
- PASS: Returns `new SomeResource($model)` or `SomeResource::collection(...)`
- WARN: Returns `$model->toArray()` or the model with `->only()`
- FAIL: Returns the Eloquent model or collection directly

**Check D — HTTP Status Codes**
Check for these specific violations:
- POST that creates → must be 201 (not 200)
- DELETE → must be 204 (not 200, unless body included)
- Validation error → must be 422 (not 400 or 200)
- Unauthenticated → must be 401 (not 403)
- Permission denied → must be 403 (not 401)

**Check E — Service Layer Separation**
- PASS: No Eloquent calls in the controller (`::find`, `::create`, `::where`, `->save`, `->update`, `->delete`)
- FAIL: Contains Eloquent queries that belong in a Service class

## Step 4: Check Route Naming

Verify:
- Every route has a name (auto-generated by `Route::apiResource()` is fine)
- Names follow `{resource}.{action}` convention
- Report any unnamed routes
